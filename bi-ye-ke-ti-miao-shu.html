毕业课题描述

Llming | 2019 Mar 02 Sat

Project Description

The World Wide Web has improved the lives of many. People with disabilities belong to the group of users that can benefit most from it. However, to a large extent, the Web is not accessible. The W3C consortium has proposed a set of guidelines that web pages should satisfy to make the websites accessible, ie the Web Content Accessibility Guidelines.

While the access to health should universal, if websites containing critical information about the services offered by the National Health Service are not accessible, a number of citizens get excluded from it.

Main purpose

The purpose of this project is to establish a software system to automatically evaluate the websites of General Practices (primary care. The next activities should be conducted to successfully complete the project.

  • Analysis of existing automatic evaluation tools in order to choose those who can be embedded in a software system.
  • Use a crawler to collect the URLs and information of the General Practices from https://www.nhs.uk/service-search/GP/Location/Places/A/4
  • Develop a system that evaluates the accessibility of those websites and stores the results in a regular fashion
  • A dashboard that monitors the accessibility of websites over time
  • An statistical analysis between compliance to accessibility guidelines, and (i) general satisfaction with the GP and (ii) use of online services by the people registered in that practice

Level

This is a personal project, so there are no other team members.

Introduce the area and Key points

In 《Web Content Accessibility Guidelines 2.0》, the description of accessibility is: the readability and comprehensibility of Web content for users with disabilities. At the same time, the guide also specifies: improving accessibility also makes it easier for ordinary users to understand web content.

Specifically, website accessibility should consider two aspects: 1. Whether the user is disabled or not, the user agent must be used to access the web content. Therefore, to improve accessibility, first consider a variety of user agents: desktop browsers, voice browsers, mobile phones, car PCs, etc.Google has hired a number of disabled employees to help improve the accessibility of the product. 2. Also consider the environmental restrictions when users access web content. For example: noisy environment, too dark or too bright room, or hands-free, etc. This is a higher level of accessibility requirements, do it, can Create the core competitiveness of products in specific areas. There are three aspects to simple summarization: coping with normal browsing needs in extreme environments; structure, performance, and behavioral cross-terminal platform compatibility support; ensuring accessibility information for people with disabilities. Web1.0 website is mainly content; Web2.0 website is mainly based on functions. At present, there are three types of web interface forms: "pure function, content + function, pure content".The pattern has been basically stereotyped. Although content-based websites are more prominent in accessibility, feature-based websites are more prominent in Usability, but it is not that feature-based websites do not need to consider accessibility.

Some idea

I personally tend to use Python to develop web crawlers for the collection of NHS website URL links. In the automatic evaluation tool, "Pa11y" has a relatively satisfactory dashboard and problem description, and it also has a relatively easy to call API. Therefore, Tentatively "Pa11y" is an evaluation tool. The developed system personally tends to use Python, in which the database can use MongDB, such No-SQL database has good compatibility with storage evaluation results.
